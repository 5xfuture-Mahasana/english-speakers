# Hostinger Node.js Deployment Guide — English Speakers

## Overview

This project uses **Next.js 15 standalone output**, which bundles the server into
`.next/standalone/server.js` — a self-contained Node.js HTTP server with zero
external runtime dependencies.

---

## 1. Build Locally (on your machine)

```bash
# 1. Install dependencies
npm ci

# 2. Build for production (one command does everything)
npm run build
```

> **Note:** Next.js 15 automatically copies `.next/static/` into the standalone
> bundle during build. No manual copy step is needed for this project.

---

## 2. Files to Upload to Hostinger

Upload **only** the following folder to your Hostinger Node.js root:

```
.next/standalone/          ← entire folder (includes server.js + node_modules)
  ├── server.js             ← Next.js built-in startup file
  ├── node_modules/         ← minimal runtime deps (auto-generated by build)
  ├── public/               ← copied in step 1c above
  └── .next/
        └── static/         ← copied in step 1c above
```

Also upload these two files to the **same root** as `standalone/`:

| File | Purpose |
|---|---|
| `server.js` | Hostinger Node.js entry point (delegates to `.next/standalone/server.js`) |
| `ecosystem.config.js` | PM2 process manager config |
| `.env.production.local` | Your real environment variables (create from `.env.production`) |

### What NOT to upload
- `node_modules/` at project root (standalone has its own trimmed copy)
- `src/` / `app/` source files
- `.env.production` template (use `.env.production.local` with real values)

---

## 3. Environment Variables on Hostinger

### Option A — Hostinger Panel (recommended)
1. Go to **hPanel → Node.js → Environment Variables**
2. Add each key/value from `.env.production`

### Option B — `.env.production.local` file
Create `.env.production.local` on the server (next to `server.js`):

```env
NEXT_PUBLIC_APP_URL=https://englishspeakers.in
NEXT_PUBLIC_WHATSAPP_NUMBER=919849668255
NODE_ENV=production
PORT=3000
NEXT_TELEMETRY_DISABLED=1
```

---

## 4. Node.js Startup Configuration on Hostinger

In **hPanel → Node.js**:

| Setting | Value |
|---|---|
| Node.js version | 20.x (LTS) |
| Application root | `/home/<user>/public_html` (or your domain folder) |
| Application startup file | `server.js` |
| Application mode | Production |

Hostinger will run: `node server.js`

## 5. PM2 (Process Manager) — Optional but Recommended

PM2 keeps the app alive across crashes and restarts the server on reboot.

### Install PM2

```bash
npm install -g pm2
```

### Start with PM2

```bash
# Start the app
pm2 start ecosystem.config.js --env production

# Save process list (survives reboots)
pm2 save

# Generate startup script — run the printed command as root
pm2 startup
```

### PM2 Commands Reference

```bash
pm2 list                        # Show all running processes
pm2 logs english-speakers       # Stream logs
pm2 restart english-speakers    # Restart app
pm2 stop english-speakers       # Stop app
pm2 delete english-speakers     # Remove from PM2
pm2 monit                       # Live CPU/memory dashboard
```

## 9. Troubleshooting

**App shows blank page or 500 error**
→ Check PM2 logs: `pm2 logs english-speakers`
→ Confirm `public/` and `.next/static/` were copied into standalone

**Static assets (images, CSS) return 404**
→ Confirm `.next/standalone/.next/static/` exists after build. If missing, re-run `npm run build`.

**Port conflict**
→ Set `PORT=<another port>` in environment variables and update hPanel

**Out of memory crashes**
→ Lower `max_memory_restart` in `ecosystem.config.js` to `300M`
→ Or upgrade to a Hostinger plan with more RAM

**WhatsApp links not working**
→ Check that your domain resolves and HTTPS is active — WhatsApp web.whatsapp.com
  only opens `wa.me` links from HTTPS origins
